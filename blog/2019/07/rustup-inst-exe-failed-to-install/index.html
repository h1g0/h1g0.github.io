[Rust] rustup-inst.exeでの新規インストール時に「アクセスが拒否されました。(os error 5)」が出る時の対処法

2019-07-28

1分で読めます

 Twitter

 Facebook

 LinkedIn

 Email

Windows 10にrustup-inst.exeを使ってRust1をインストールしようとしたところ、アクセスが拒否されました。(os error 5)のエラーが出てインストールできなかった。それを解決した際のメモ。

なお、インストールしようとしたrustupのバージョンは

rustup 1.18.3 (435397f48 2019-05-22)

である。

解決法

とりあえずOSの再起動等を試してみても効果がなかったのでググってみたところ、githubのプロジェクトページに

fresh install fails on Windows: &ldquo;error: could not rename component file from &lsquo;&hellip; ust-docs\share/doc/rust/html&rsquo; &hellip;&quot;

というIssueが上がっているのを見つけた。読んでみたところ、自分の問題と同じであった。

この記事を参考にアンチウィルスソフト（自分の場合はAvira）のリアルタイムプロテクションを一時的に切ってrustup-inst.exeを再試行したところ、今度は正常にインストールできた。

以前rustup-inst.exeを使って同環境でセットアップを行った際には問題は発生しなかったので、rustup-inst.exeのアップデートによるものかAviraの更新に起因する問題なのかもしれない。
